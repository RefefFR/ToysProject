Toy API Documentation

Documentation for the Toy API endpoints.

Base URL

        http://localhost:3001/toys

Endpoints

GET Requests

Get All Toys

        GET /toys

Returns a list of all toys.

Example Request:

GET http://localhost:3001/toys

Example Response:

{
    "msg": "toys work!"
}

Search by Name or Info

        GET /toys/specify/?name={name}&info={info}

Search for toys by name or info.

Example Request:

GET http://localhost:3001/toys/specify/?name=Toy1&info=Info1

Example Response:

[
    {
        "_id": "123456",
        "name": "Toy1",
        "info": "Info1"
    }
]

Get Toys by Category

        GET /toys/category/?catname={categoryName}

Get toys by category.

Example Request:

GET http://localhost:3001/toys/category/?catname=Category1

Example Response:

[
    {
        "_id": "123456",
        "name": "Toy2",
        "category": "Category1"
    },
    {
        "_id": "789012",
        "name": "Toy3",
        "category": "Category1"
    }
]

Filter by Price Range

        GET /toys/prices/?min={minPrice}&max={maxPrice}

Filter toys by price range.

Example Request:

GET http://localhost:3001/toys/prices/?min=20&max=50

Example Response:

[
    {
        "_id": "123456",
        "name": "Toy4",
        "price": 30
    },
    {
        "_id": "789012",
        "name": "Toy5",
        "price": 40
    }
]

Get Toy by ID

        GET /toys/{id}

Get a toy by its ID.

Example Request:

GET http://localhost:3001/toys/123456

Example Response:

{
    "_id": "123456",
    "name": "Toy6",
    "info": "Info6"
}

PUT Requests

Update a Toy

        PUT /toys/{id}

Update a toy by ID. Requires authentication.

Example Request:

PUT http://localhost:3001/toys/123456

Example Response:

{
    "message": "Toy updated successfully"
}

POST Requests

Create a New Toy

        POST /toys

Create a new toy. Requires authentication.

Example Request:

POST http://localhost:3001/toys

Example Response:

{
    "_id": "123456",
    "name": "New Toy",
    "info": "New Toy Info",
    "user_id": "789012"
}

DELETE Requests

Delete a Toy

        DELETE /toys/{id}

Delete a toy by ID. Requires authentication.

Example Request:

DELETE http://localhost:3001/toys/123456

Example Response:

{
    "message": "Toy deleted successfully"
}

Authentication

Endpoints d, e, and f require authentication. To authenticate, include a valid token in the request headers.
